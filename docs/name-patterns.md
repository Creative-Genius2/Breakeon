## EXTRACTED PATTERNS (in priority order)

### Pattern 1: "Series Name: Subtitle" → "Subtitle"
**This is by far the dominant pattern.** When a game has a clear series prefix separated by a colon (or dash), people drop the series name entirely and refer to the game by its subtitle alone. This works because within any conversation, the series context is already established.

Examples: Zelda, Castlevania, Fire Emblem, Kirby, Kingdom Hearts, Phoenix Wright, Professor Layton, Mario & Luigi, Mega Man

**Rule: If the title contains ":" or " - ", everything before the separator is a candidate for removal.**

### Pattern 2: "Series [Descriptor] Version/Edition" → "[Descriptor]"
**Specific to Pokémon but the principle applies elsewhere.** When "Version" or "Edition" appears, always drop it. The descriptor word IS the name.

**Rule: Always strip "Version", "Edition" from the end.**

### Pattern 3: Leading articles get dropped from subtitles
"The Legend of Zelda: **The** Minish Cap" → "Minish Cap"
"Fire Emblem: **The** Sacred Stones" → "Sacred Stones"
"Kirby & **the** Amazing Mirror" → "Amazing Mirror"
"Yu-Gi-Oh! **The** Sacred Cards" → "Sacred Cards"
"Professor Layton and **the** Curious Village" → "Curious Village"

**Rule: Strip "The ", "A ", "An " from the beginning of the resulting name after other patterns have been applied.**

### Pattern 4: Short/iconic series names are kept
When the full title is already short and the series name IS the identity:
- "Sonic Rush" stays "Sonic Rush" (removing "Sonic" leaves just "Rush" — too generic)
- "Mario Kart DS" stays "Mario Kart DS"
- "Advance Wars" stays "Advance Wars"
- "Pokémon Ranger" stays "Pokémon Ranger"

**Rule: If removing the series prefix leaves a word that is < 2 words and generic (Rush, Battle, Zero, Party), keep the series name.**

### Pattern 5: Numbered sequels abbreviate
- Final Fantasy IV → FF4
- Dragon Quest IX → DQ9
- Mega Man Zero 2 → MMZ2 or Zero 2
- Advance Wars 2 → AW2

**Rule: "[Series] [Roman/Arabic numeral]" → abbreviated. But this is mostly for typed shorthand, not display names.**

### Pattern 6: Multi-level subtitle chains → keep deepest distinctive part
"Phoenix Wright: Ace Attorney - Justice for All" has THREE levels:
1. Character name (Phoenix Wright)
2. Series name (Ace Attorney)  
3. Specific game (Justice for All)

People use level 3.

"Yu-Gi-Oh! 5D's World Championship 2009: Stardust Accelerator" has FOUR levels:
1. Franchise (Yu-Gi-Oh!)
2. Series tag (5D's)
3. Sub-series (World Championship 2009)
4. Subtitle (Stardust Accelerator)

People use level 3 (the year is more distinctive than the subtitle).

**Rule: For deeply nested titles, find the most distinctive + shortest identifying segment.**

### Pattern 7: "And the" / "And" connectors get stripped
"Professor Layton **and the** Curious Village" → "Curious Village"
"Kirby **& the** Amazing Mirror" → "Amazing Mirror"
"Mario **&** Luigi: Superstar Saga" → series name gets dropped via Pattern 1

**Rule: Strip "and the", "& the", "and" when they connect a character name to a subtitle.**

### Pattern 8: Remakes/ports reference the original
"Super Mario Advance 4: Super Mario Bros. 3" → "Mario Bros. 3" or "SMB3"
People use the name of the ORIGINAL game, not the port label.

**Rule: If a title contains both a port/collection name AND an original game title, use the original game title.**

### Pattern 9: Platform suffixes are context-dependent
"DS", "Advance", "3D" suffixed to titles:
- "Mario Kart DS" → kept because there are many Mario Karts
- "Super Mario 64 DS" → "Mario 64 DS" or just "SM64DS"
- "Sonic Advance" → kept because "Advance" is the game's identity

**Rule: Platform suffixes are kept when needed for disambiguation within the series. Dropped when the context is obvious.**

### Pattern 10: "the [Character]" noise in numbered titles
When a series name includes a character descriptor that nobody uses in speech:
- "Sonic the Hedgehog" → "Sonic" ("the Hedgehog" is logo text, not a spoken name)
- "Sonic the Hedgehog 2" → "Sonic 2"
- "Sonic the Hedgehog 3" → "Sonic 3"
- "Sonic the Hedgehog" (no number) → "Sonic 1" (number added to distinguish from the series)

**Rule: Strip "the [Character]" from series names when the descriptor is decorative. Add "1" when stripping leaves a bare name with no number.**

### Pattern 11: Lock-on ROM titles — detect `+` and flip order
Lock-on ROMs are named with the add-on cart first: `"Sonic & Knuckles + Sonic the Hedgehog 3"`. But the community name puts the base game first: **"Sonic 3 & Knuckles"**.

- `Sonic & Knuckles + Sonic the Hedgehog 3` → **Sonic 3 & Knuckles**
- `Sonic & Knuckles + Sonic the Hedgehog 2` → **Sonic 2 & Knuckles**

**Rule: If `+` appears and one side is "Sonic & Knuckles", extract the number from the other side and output "[Sonic N] & Knuckles". Always flip to base-game-first order.**

**NOTE: Lock-on filenames vary by dumper. Common forms include `+` and `&` as separators. The presence of "Sonic & Knuckles" alongside another Sonic title is the reliable signal.**

## IMPLEMENTATION PRIORITY

For an auto-namer, apply in this order:
1. Strip region/dump tags: (USA), (Europe), [!], etc.
2. Strip "Version", "Edition"
3. Strip "The Legend of Zelda: " / other known long series prefixes
4. Apply "Series: Subtitle" → "Subtitle" if colon/dash present
5. Strip leading "The ", "A " from result
6. Check if result is too short/generic → if so, restore series name
7. Title-case the result
8. Check override file for exact match (handles weird cases)
